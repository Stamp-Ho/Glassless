# 05. 보안 및 운영 제약 체크리스트

## 1) 환경변수 규칙

- `OPENAI_API_KEY`, `DATABASE_URL` 하드코딩 금지
- 설정은 `pydantic-settings` 또는 `os.getenv` 사용
- `.env`는 저장소 커밋 금지

## 2) 인증/인가 단순화 규칙

- JWT, OAuth, 세션 인증 도입 금지 (MVP 범위 외)
- 게시글 수정/삭제 시 평문 비밀번호 비교 사용
- 비밀번호 해시 제안/적용 금지 (명시 요구사항)

## 3) API 비용 통제

- OpenAI 요청마다 `max_tokens` 강제
- 요청 타임아웃 강제
- 과도한 입력 길이 제한
- 컨텍스트 문서 개수 상한 설정

## 4) 예외 처리

- 입력 오류: 422
- 권한(비밀번호 불일치): 403
- 리소스 없음: 404
- 외부 API 실패: 502 또는 503

## 5) PR 전 셀프 점검

- 민감정보 하드코딩 없음
- `.env` 미포함
- 비동기 엔드포인트 사용
- 요청/응답 스키마 선언
- Swagger(`/docs`)에서 스키마 노출 확인
