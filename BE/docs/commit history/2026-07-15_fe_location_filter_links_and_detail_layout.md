# 2026-07-15 작업 요약: 게시글 위치 필터 라우팅, 명소 링크, 상세 버튼 레이아웃 개선

## 개요
게시글 목록/상세 화면에서 명소 선택 흐름을 강화하고, URL 기반 필터 유지와 외부 지도 이동/동일 명소 게시글 모아보기 동선을 추가했다. 또한 상세 화면의 액션 버튼 레이아웃을 모바일에서도 PC와 유사하게 보이도록 조정했다.

## 변경 사항

### 1) 상세 경로 정리
- 상세 라우트를 `/posts/:id`로 통일
- 기존 `/post/:id`는 alias로 유지하여 기존 링크 호환

### 2) location_id 필터의 라우터 쿼리 동기화
- `/posts?location_id=...`로 유입 시 목록 필터 자동 적용
- URL에서 `location_id`가 제거되면 필터도 자동 해제
- 목록에서 상세 이동 시 현재 `location_id` 쿼리 전달
- 상세에서 목록 복귀/삭제 후 이동 시 `location_id` 쿼리 유지

### 3) 명소 카드 링크 확장
- 선택된 명소 영역에 카카오맵 이동 링크 추가
- 링크 생성 우선순위:
  1. 주소 기반 검색
  2. 좌표(mapx/mapy) 기반 이동
  3. 이름 검색
- 카카오 링크 왼편에 `게시글 모아보기` 버튼 추가
  - 이동 경로: `/posts?location_id={현재 명소 id}`

### 4) DetailView 명소 정보 노출 강화
- 읽기 모드에서 `location_id`가 존재하면 연결 명소 카드 노출
- 명소명/권역/카테고리/주소와 링크 버튼 표시

### 5) DetailView 액션 버튼 레이아웃 정리
- 수정/삭제, 취소/수정 확정 버튼 그룹을 클래스 기반으로 재정리
- 모바일에서도 가로 정렬 유지(`nowrap`) 및 버튼 축소로 PC와 유사한 배치 유지
- 기존 인라인 스타일 제거
