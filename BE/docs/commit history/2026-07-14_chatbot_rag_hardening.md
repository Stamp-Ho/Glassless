# 2026-07-14 챗봇 RAG 하드닝

## 작업 개요

챗봇이 사용자 질문에 답할 때 반드시 로컬 데이터 범주 내에서만 답하도록 프롬프트와 검색 흐름을 강화했습니다.

## 변경 파일

- `BE/src/app/routers/chat.py`

## 상세 변경

1. RAG 후보 검색 보강
- `Location` 테이블에서 질문 키워드/권역/카테고리 기반 후보 검색
- 후보에 명소 별점 평균/개수까지 함께 계산해 컨텍스트에 포함

2. 시스템 프롬프트 강화
- 외부 지식, 추측, 일반 상식, 인터넷 정보 사용 금지
- 참고 데이터에 없는 내용은 명시적으로 답변 불가 처리
- 답변은 한국어로 짧고 명확하게 작성

3. 응답 정책
- 후보가 없으면 로컬 데이터로 답할 수 없다는 문구를 반환
- OpenAI 호출 시 `max_tokens`와 `timeout` 유지

## 결과

- 챗봇이 `Location` 기반 RAG 흐름으로만 답변하도록 제한됨
- FE는 이 API를 사용해 로컬 데이터 내 질의응답만 유도할 수 있음
